Steps to Clone a Repo Using Eclipse:

  1. Open Eclipse.

  2. Configure the window to show Git Views. To do this, go to Window/Show View/Other/Git. Highlight all/Ok. Package Explorer should be at top left, Git Repos at bottom left, other git tabs bottom right below the editing window.

  3. Choose Clone a Git Repository in the Git Repositories view (icon with the blue arrow).

  4. Open your GitHub account and choose the repository you want to clone. Copy the HTTPS clone URL (should look something like https://github.com/yourname/reponame).

  5. In the Clone Git Repository dialog box in Eclipse, paste the URL you copied. Note that the Host and Repository fields self populate.

  6. Enter your GitHub user name and password. Check the box next to Store in Secure Store. Click Next.

  7. In the Branch Selection, make sure both boxes have check marks. Click Next.

  8. Under Local Destination, browse to the GitHub workspace folder on your local drive (or create one) to switch to the appropriate directory. Double click the workspace folder. Click Save. Click Finish (perhaps now would be a good time to begin praying). Wait for the operation to complete.

  9. Click No in the Secure Storage dialog box to decline password recovery. (note that if you're using Eclipse, the program automatically restarts somewhere in here to reset itself with your correct info.)

  10. Right click in Project or Package Explorer window. Choose New/Project. Open General and select Project. Click Next. Give the Project a name and click Finish.

Commit:

  1. Click on the Git Staging tab in Eclipse. A list of new files should appear in the Unstaged Changes area.

  2. Highlight the files you wish to commit and drag them to the Staged Changes area.

  3. Add a Commit Message (this tells other users about your file if new, and about changes if recently updated). Click Commit and Push to send the files to your GitHub repository.
